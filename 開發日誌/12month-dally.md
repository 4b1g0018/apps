# 12月 專題最終開發總結 (完整記錄)

## 1. 最終衝刺功能更新 (Final Sprint)
在此階段，我們補足了社群功能的最後一塊拼圖，並進行了程式碼清理與使用者體驗優化。

### 社群互動優化
*   **貼文日期補登功能 (Date Picker)**
    *   在 `CreatePostPage` 新增日期選擇器，打破原本只能發布「當下」時間貼文的限制。
    *   使用者現在可以補登過去日期的訓練紀錄或成果，讓訓練日誌更完整。
*   **貼文刪除功能 (Delete Post)**
    *   在 `PostDetailPage` 實作刪除邏輯。
    *   加入權限判斷：只有貼文作者本人能看到刪除按鈕。
    *   加入 iOS 風格確認對話框 (Alert Dialog) 與操作結果提示 (SnackBar)，提升互動質感。

### 介面與資料簡化
*   **移除冗餘欄位**
    *   移除 `CommunityProfilePage` 中未實際使用的「家鄉 (Hometown)」欄位。
    *   清理相關的編輯 UI 與邏輯，讓個人檔案頁面更專注於訓練數據與貼文展示。

### 系統穩定性與 Bug 修復
*   **雲端還原邏輯修正**
    *   修復 `SettingsPage` 中「從雲端還原資料」功能在使用者點擊「取消」後仍會錯誤執行的嚴重 Bug。
*   **Git 版控問題解決**
    *   重新初始化專案 Git Repository，解決目錄結構錯誤導致無法 Push 至 GitHub 的問題。
    *   修正 Remote URL，確保最終程式碼能完整上傳。

## 2. 核心架構回顧 (12/2 重點摘要)
回顧本月稍早的重大更新，我們完成了單機版到雲端版的轉型：

### 雲端架構整合 (Backend & Cloud)
*   **Firebase Auth**：實作 Email/Password 註冊登入，並解決與本地 SQLite 模型的衝突。
*   **雙軌登入策略**：新裝置登入自動拉取雲端備份，實現無縫換機。
*   **Firestore 資料庫**：
    *   `users`：同步個人生理數據。
    *   `posts`：儲存社群貼文 (Base64 圖片存儲)。
    *   `workout_logs`：訓練紀錄自動備份。

### 社群功能 (Community Features)
*   **介面重構**：改版為黑色系 Retro 風格，實作置頂週曆時間軸與日期篩選。
*   **圖片處理**：整合 image_picker 並實作圖片壓縮 (Base64)，解決 Firestore 儲存限制。
*   **好友系統**：實作 Email/暱稱搜尋與唯讀權限控制。

### 訓練體驗 (User Experience)
*   **一鍵開始**：首頁自動偵測今日課表。
*   **智慧排序**：執行中的動作自動排至首位。
*   **圖表修復**：修正體重趨勢圖的 X 軸顯示問題。

---

## 3. 專題最終結論 (Project Conclusion)
此專題透過 Flutter 框架開發，便於整合 iOS 與 Android 雙系統。
系統透過手機鏡頭整合 OpenPose 輔助動作分析，並結合穿戴裝置即時監測心率與熱量，實現科學化的訓練紀錄；
旨在協助自主健身者紀錄長期變化，並改善因缺乏專業指導而導致的姿勢問題。
